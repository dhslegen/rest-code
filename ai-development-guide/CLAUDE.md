# 项目：AI辅助开发实战指南

## 项目概述
这是一个系统性的知识分享项目，旨在帮助开发者掌握AI辅助编程的核心方法论。内容涵盖LLM基础、主流工具（Cursor、Claude Code）使用、实战案例和最佳实践。

## 项目类型
- **类型**: 文档项目（Documentation Project）
- **格式**: Markdown
- **组织**: 模块化章节结构
- **目标**: 知识分享和教学

## 内容规范

### 写作风格
- **语气**: 专业但易懂，贴近实战
- **视角**: 基于个人经验的分享，真实可信
- **结构**: 从理论到实践，循序渐进
- **示例**: 提供具体可操作的案例

### Markdown规范
- 标题层级：最多使用三级标题 `###`
- 代码块必须指定语言: \`\`\`javascript
- 列表保持一致缩进（2空格或4空格）
- 中英文之间添加空格: `这是 Token 的概念`
- 使用相对路径引用内部资源: `[链接](./chapters/01-llm-fundamentals.md)`
- 表格对齐: 使用 `|--|` 分隔符

### 内容质量要求
- ✅ **准确性**: 所有技术信息必须准确，可验证
- ✅ **实用性**: 提供可直接应用的方法和技巧
- ✅ **完整性**: 每个主题有完整上下文，不遗漏关键信息
- ✅ **可读性**: 使用图表、表格、代码块增强可读性
- ✅ **真实性**: 案例和经验必须真实，不夸大

## AI辅助开发工作流

### 内容创作流程
1. **理解主题**:
   - 先阅读 `OUTLINE.md` 了解章节定位
   - 查看上下文和章节间的逻辑关系

2. **检查进度**:
   - 查看 `TODO.md` 确认当前待完成任务
   - 确认任务优先级

3. **内容创作**:
   - 根据提纲创作章节详细内容
   - 保持与总体风格一致

4. **质量检查**:
   - 运行质量检查清单（见下方）
   - 确保符合所有规范

5. **更新记录**:
   - 更新 `TODO.md` 标记任务完成
   - 在 `CHANGELOG.md` 记录重要变更

### 章节创作要求

每个章节文件应包含：

```markdown
# 第X章：章节标题

> 简短的引言，说明本章目标和重要性

## X.1 第一节标题

### X.1.1 小节标题
内容...

**要点总结**:
- 关键点1
- 关键点2

---

## 章节小结
总结本章核心内容

## 延伸阅读
- [资源1](URL)
- [资源2](URL)
```

### 代码示例要求

所有代码示例必须：
- 指定语言
- 提供注释
- 可直接运行（或明确标注为伪代码）
- 附带说明

示例格式：

````markdown
\`\`\`javascript
// 清晰的注释说明代码作用
function calculateTokens(text) {
  // 简化示例：实际Token计算更复杂
  return Math.ceil(text.length / 4);
}

console.log(calculateTokens("Hello, AI!"));  // 输出: 3
\`\`\`

**说明**: 这是一个简化的Token计算示例，实际的Token化过程由LLM的tokenizer完成，不同模型的规则可能不同。
````

### 截图和图表要求
- 截图存放在 `assets/screenshots/` 目录
- 图表优先使用Markdown表格或Mermaid
- 所有图片必须有 alt 文本
- 控制图片大小，避免过大文件

- ❌ **不要创建空章节**: 只有准备好完整内容时才创建文件
- ❌ **不要复制未验证内容**: 所有代码和技术信息必须验证
- ❌ **不要使用过时信息**: 工具版本、API等必须是最新的
- ❌ **不要遗漏更新**: 完成任务后必须更新TODO和CHANGELOG
- ❌ **不要夸大**: 保持真实，不夸大AI工具的能力
- ❌ **不要忽略细节**: 配置步骤、命令等必须准确
